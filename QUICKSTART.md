## 🚀 Quick Start (5 Minutes)

### Step 1: Configure Email (2 min)
- [ ] Open `.env` file in the project root
- [ ] Fill in these 5 lines:
  ```bash
  SMTP_HOST=smtp.gmail.com
  SMTP_PORT=587
  SMTP_USER=your-email@gmail.com
  SMTP_PASS=your-app-password  # Not your regular password!
  SMTP_FROM=your-email@gmail.com
  ```
  > For Gmail: Get app password at https://myaccount.google.com/apppasswords
  > For other providers: See EMAIL_SETUP.md

### Step 2: Test Configuration (2 min)
- [ ] Open terminal, navigate to project
  ```bash
  cd legal-case-picker/server
  npm install  # If first time
  node test-email.js
  ```
- [ ] Look for: `✅ Test email sent successfully!`
- [ ] Check your inbox for test email

### Step 3: Start Server (1 min)
- [ ] In same terminal:
  ```bash
  npm run dev
  ```
- [ ] Server should start on http://localhost:5001

## 📧 Using Email Feature

### Send Single Email
1. Go to Notices Dashboard
2. Find a "Signed" notice
3. Click the email icon in actions
4. Email sends with PDF to borrower
5. Notice status changes to "Dispatched"

### Send Bulk Emails
1. Go to Notices Dashboard
2. Filter by "Signed" status
3. Select multiple notices (checkboxes)
4. Click "Email" button in toolbar
5. All selected notices get emailed

## 🐛 Troubleshooting

### Email Config Test Fails
```bash
Error: SMTP connection failed
```
**Fix:**
- Check SMTP_USER and SMTP_PASS in .env
- If Gmail: Use app password, not regular password
- Gmail: Ensure 2FA is enabled
- Check port: port 587 works better than 465

### Test Email Doesn't Arrive
- [ ] Check spam/junk folder
- [ ] Verify SMTP_FROM is correct
- [ ] Check mail server logs with provider

### "No email address available for this borrower"
- [ ] Check CSV has "Email" column
- [ ] Verify case/borrower record in database has email
- [ ] Case picker must load data first

### PDF Attachment Not In Email
- [ ] Check Puppeteer is working (`pdfService.js`)
- [ ] Check server logs for PDF generation errors
- [ ] Verify notice\_content is valid HTML

### OpenAI Errors
- [ ] Check OPENAI_API_KEY in .env
- [ ] Verify API key has credits
- [ ] Check OpenAI rate limits

## 💡 Pro Tips

1. **Test Emails to Yourself First**
   - Set SMTP_FROM to your test email
   - Send notices to yourself before borrowers
   - Verify PDF, subject, body formatting

2. **Monitor Email Sending**
   - Check server logs: `npm run dev`
   - Notice status updates to "dispatched"
   - Track dispatch_email in database

3. **Bulk Sending Strategy**
   - Start with small batches (10-20)
   - Monitor delivery rates
   - Gradually increase batch size

4. **Personalization**
   - Email body automatically uses borrower name
   - Loan number in subject line
   - Outstanding amount in email body
   - All from case data

5. **Recovery from Failures**
   - Failed emails show error
   - Can retry individual notices
   - Check database for dispatch_email
