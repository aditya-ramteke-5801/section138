# Email Configuration Guide

## Overview

The email feature is **fully implemented** and ready to use! The system:
- Generates dynamic email subjects and bodies using OpenAI
- Attaches the generated legal notice as PDF
- Sends from a hardcoded email address to borrower emails
- Supports both single and bulk email sending

## Email Service Architecture

### Components
1. **emailService.js** - Nodemailer transporter setup and mail sending
2. **noticeService.js** - OpenAI-powered email content generation
3. **Backend endpoints** - `/api/notices/:id/send-email` and bulk variant
4. **Frontend** - Dashboard "Email" button for signed notices

### Email Generation Flow
```
Click Send Email → Generate PDF → Generate Subject (OpenAI) 
→ Generate Body (OpenAI) → Send via SMTP → Update status to "dispatched"
```

## Setup Instructions

### Step 1: Choose Your Email Provider

#### Option A: Gmail (Recommended for Testing)

1. Go to [https://myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
2. You must have 2-factor authentication enabled
3. Select "Mail" and "Windows Computer" (or your device)
4. Google generates a 16-character app password
5. In `.env`, set:
   ```
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_USER=your-email@gmail.com
   SMTP_PASS=xxxx-xxxx-xxxx-xxxx
   SMTP_FROM=your-email@gmail.com
   ```

#### Option B: AWS SES (Production)
- Enables sending from your domain
- More reliable for business use
- Set in `.env`:
  ```
  SMTP_HOST=email-smtp.REGION.amazonaws.com
  SMTP_PORT=587
  SMTP_USER=your-ses-smtp-user
  SMTP_PASS=your-ses-smtp-password
  SMTP_FROM=noreply@yourdomain.com
  ```

#### Option C: SendGrid
- Simple setup, no domain verification needed initially
- Set in `.env`:
  ```
  SMTP_HOST=smtp.sendgrid.net
  SMTP_PORT=587
  SMTP_USER=apikey
  SMTP_PASS=SG.your_sendgrid_api_key
  SMTP_FROM=noreply@yourdomain.com
  ```

#### Option D: Your Own SMTP Server
- Configure with your provider's SMTP details

### Step 2: Update `.env` File

Edit `legal-case-picker/.env` and add the email configuration:

```bash
# Email Configuration (SMTP)
SMTP_HOST=smtp.gmail.com          # Your SMTP server
SMTP_PORT=587                     # Usually 587 (TLS) or 465 (SSL)
SMTP_USER=your-email@gmail.com    # Your email account
SMTP_PASS=app-password-here       # App password or SMTP password
SMTP_FROM=sender@yourdomain.com   # Email address shown as "From"
```

### Step 3: Verify Configuration

Restart the server:
```bash
cd legal-case-picker/server
npm run dev
```

Check the console output for any SMTP connection errors.

## Using the Email Feature

### Single Email (From Notice Editor)
1. Open a notice with status "Signed"
2. Look for email action button
3. Click to send email with PDF attachment to borrower's email

### Bulk Email (From Dashboard)
1. Go to Notices Dashboard
2. Filter by status "Signed"
3. Select multiple notices (checkboxes)
4. Click "Email" button
5. System sends to all borrowers in batch

### What Gets Sent

**Email Structure:**
- **From:** SMTP_FROM address (configured in `.env`)
- **To:** Borrower's email from the case
- **Subject:** "Legal Notice — Loan Account {loanNumber} — Immediate Action Required"
- **Body:** 4-6 lines summarizing the key points (generated by OpenAI)
- **Attachment:** PDF of the signed legal notice

**Email Body Example:**
```
Dear Borrower,

We write to you on behalf of Slice (NorthStar Fintech Pvt. Ltd.) regarding 
your outstanding loan obligation of ₹50,000 for account LN-XXXXX.

Despite repeated attempts to contact you, payment has not been received. 
You are hereby called upon to settle all outstanding amounts within 7 days 
from the date of this notice.

Please contact our office at 9187525893 (10 AM - 6 PM, working days) for 
any assistance or payment arrangements.

The full legal notice is attached as a PDF document.

Best regards,
Slice Legal Team
```

## Troubleshooting

### Error: "No email address available for this borrower"
- The borrower record must have an email field
- Check that your CSV has an "Email" or "email" column
- Verify case data is loaded correctly

### Error: "SMTP connection failed"
- Verify SMTP_HOST and SMTP_PORT in `.env`
- Check SMTP_USER and SMTP_PASS are correct
- For Gmail: ensure you used an App Password, not your regular password
- For firewall: port 587 (TLS) is usually open; 465 (SSL) might be blocked

### Error: "Mail delivery failed"
- Check recipient email is valid in borrower record
- Verify your "From" address is approved by the SMTP provider
- Check SMTP provider's bounce/delivery logs

### Emails not being sent (status not updating to "dispatched")
- Check server logs for detailed error messages
- Test SMTP credentials manually: `telnet smtp.gmail.com 587`
- Verify OPENAI_API_KEY is valid (needed for subject/body generation)

## Security Best Practices

1. **Never commit .env to git** - It's in `.gitignore`
2. **Use app-specific passwords** - Not your regular password
3. **Set up SPF/DKIM/DMARC** - If sending from your domain (AWS SES, SendGrid)
4. **Rate limiting** - Consider implementing rate limits for bulk sends
5. **Audit logs** - Track who sent what notice to whom

## API Endpoints Reference

### Send Single Email
```bash
POST /api/notices/:id/send-email
Body: { "email": "borrower@example.com" }  # Optional, uses borrower_email if not provided
Response: { success: true, notice: {...}, email_sent_to: "..." }
```

### Send Bulk Emails
```bash
POST /api/notices/bulk-send-email
Body: { "notice_ids": ["id1", "id2", "id3"] }
Response: { 
  results: [ { id: "...", success: true }, ... ],
  sent: 3,
  failed: 0
}
```

## Cost Considerations

- **Gmail:** Free, but limited to ~500 emails/day
- **AWS SES:** $0.10 per 1,000 emails (very cheap)
- **SendGrid:** Free tier: 100 emails/day; paid from $0.0001/email
- **Own SMTP:** Depends on your infrastructure

For production bulk sending, **AWS SES is recommended** (most cost-effective).
